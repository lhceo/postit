# ✦ BrainstormAI — MVP 要件定義書
Version 1.0 | 2026年3月

---

## 1. プロダクト概要

BrainstormAIは、チームが気軽にアイデアを出し合えるオンラインブレストツールです。参加の摩擦をゼロにし、AIが自動でアイデアを整理・分類・サマリー化することで、**「ブレストの場を活性化する」** ことを最大の差別化軸とします。

### コンセプト

| 項目 | 内容 |
|------|------|
| プロダクト名 | BrainstormAI（仮） |
| ターゲットユーザー | 少人数〜中規模チーム（2〜15名）、グローバル対応 |
| 主な利用シーン | 会議中のリアルタイムブレスト、非同期のアイデア収集 |
| 差別化の核心 | 参加しやすさ・アイデアが出る場の活性化 |
| 競合との違い | シンプルさ × 構造化ブレスト × 自然言語AIクエリ |

---

## 2. MVP方針

「**来やすく・始めやすく・言いやすい**」の3つを最優先に構築します。URLを踏んでから最初のアイデアを投稿するまでの **30秒以内** の体験を徹底的に磨きます。

### 開発フェーズ

| フェーズ | 内容 | 優先度 |
|---------|------|--------|
| Phase 1 (MVP) | ボード・ポストイット・カテゴリー・AI機能・場の活性化機能 | 🟢 最優先 |
| Phase 2 | 投稿リッチ化（画像・URL・テキストフォーマット） | 🟡 次フェーズ |
| Phase 3 | 外部連携・エンタープライズ機能（Google Drive等） | 🔵 将来対応 |

## 3. 機能要件

### 3-1. 参加体験（最重要）

参加の摩擦をゼロにすることが最優先。

| 優先度 | 機能 | 詳細 |
|--------|------|------|
| 🥇 1位 | URLタップだけで参加 | アカウント登録不要。ニックネームのみ入力して即参加 |
| 🥈 2位 | 匿名モード | 誰が書いたかわからないモード。心理的安全性を確保 |
| 🥉 3位 | ホストがお題・問いを提示 | ブレスト開始時にテーマ・問いをボード上部に固定表示 |
| 4位 | AIが最初の1枚を提案 | 空白ボードへの恐怖を解消。AIがお題に沿ったアイデアを1枚生成 |
| 5位 | 盛り上がりを伝えるアニメーション | 投稿・いいね時にリアクションアニメーションを表示 |
| v2 | AIが未出の視点をサジェスト | セッション中にAIがリアルタイムで不足視点を提案 |

### 3-2. ブレストボード

- ポストイットの作成・編集・削除
- ドラッグ&ドロップで自由に移動
- リアルタイム同期（同期型）＋非同期でも使用可能
- いいね・投票機能
- ダーク／ライトモード切り替え
- ドットグリッドのキャンバス表示

### 3-3. カテゴリー機能

- ホストがセッション前にカテゴリーを事前設定
- 参加者は投稿時にカテゴリーを選択
- カテゴリー別に色分け表示
- カテゴリー別エリア（レーン）でボードを区切り表示
- カテゴリーでフィルタリング

### 3-4. AI機能

| 機能 | 説明 | 実行タイミング |
|------|------|--------------|
| 自動グルーピング | 似たアイデアをテーマ別に分類 | 手動ボタン / 一定数で自動 |
| カテゴリーラベル提案 | グループ名をAIが命名 | 手動ボタン / 一定数で自動 |
| 全体サマリー | ブレスト全体を100字程度でまとめる | 手動ボタン / 一定数で自動 |
| 自然言語クエリ | 「色に関するアイデアをまとめて」など自由に質問 | 手動（任意のタイミング） |
| 最初の1枚提案 | お題に沿ったアイデアをAIが生成 | セッション開始時 |

### 3-5. セッション管理

- URLシェアだけで参加できる（登録不要）
- ホストが招待リンクを発行する方式
- ログイン必須の招待制（セキュリティが必要な場合）
- セッションの開始・終了・一時停止のコントロール
- 参加者の状態確認（誰が参加中か）
- 不適切な投稿の非表示・削除

### 3-6. 認証

- ホストはアカウントログイン必須
- 参加者はゲスト参加可（ニックネームのみ）
- ゲスト・ログイン済みの両パターンをサポート

---

## 4. ホスト管理・ダッシュボード

### 4-1. セッション中のダッシュボード（リアルタイム）

- 参加者数のリアルタイム表示
- 投稿数の推移グラフ
- 人気アイデアランキング（いいね数順）
- カテゴリー別の投稿分布
- セッション中 / 終了後でビューを切り替え可能

### 4-2. セッション履歴・一覧

- 過去のセッション一覧・履歴管理
- セッション個別の詳細分析ページ
- 参加者数・投稿数・アイデアランキングの確認

### 4-3. アウトプット・エクスポート

| アウトプット | 形式 | 優先度 |
|------------|------|--------|
| 全投稿テキスト | テキスト / CSV | 🟢 MVP必須 |
| 指定サマリーテキスト | テキスト | 🟢 MVP必須 |
| AIサマリー付き議事録 | 自動生成テキスト | 🟢 MVP必須 |
| 投票結果・アイデアランキング | テキスト / CSV | 🟡 Phase 2 |
| ボードのスナップショット | PNG / PDF | 🟡 Phase 2 |
| PDFスライド形式 | PDF | 🔵 Phase 3 |

## 5. UI/UXデザイン方針

| 方針 | 内容 |
|------|------|
| テーマ | ダーク／ライトモード切り替え対応 |
| デザイン方向性 | グローバルで通用するミニマル×プレミアム |
| フォント | DM Sans（英数）＋ Hiragino Kaku Gothic（日本語） |
| カラーアクセント | インディゴ（#6366F1）をメインアクセントに |
| アニメーション | 投稿・いいね時のマイクロインタラクション実装 |
| 対応デバイス | PC・スマートフォン（iPhone含む）完全対応 |
| 参加摩擦 | URLタップ→投稿まで30秒以内を目標 |

---

## 6. 非機能要件

| 項目 | 要件 |
|------|------|
| スケーラビリティ | 少人数（2名）〜大人数（100名以上）まで対応できる設計 |
| リアルタイム性 | 投稿・いいねが全参加者に即時反映（WebSocket等） |
| 対応言語 | 日本語・英語（グローバル対応を想定） |
| セキュリティ | セッションURL推測困難なトークン設計、ゲスト権限の制限 |
| パフォーマンス | 初回ロード3秒以内、投稿反映1秒以内 |

## 7. 技術スタック（推奨）

| レイヤー | 技術候補 | 理由 |
|---------|---------|------|
| フロントエンド | Next.js (React) + Tailwind CSS | 高速開発・SEO対応・デプロイ容易 |
| リアルタイム同期 | Supabase Realtime | WebSocketベース、スケーラブル |
| バックエンド | Next.js API Routes + Supabase | サーバーレスで管理コスト低 |
| データベース | Supabase (PostgreSQL) | リアルタイム機能内蔵 |
| AI機能 | Claude API (claude-sonnet-4-20250514) | グルーピング・サマリー・クエリ |
| 認証 | Supabase Auth | ゲスト・ログインの両対応 |
| デプロイ | Vercel | Next.jsとの親和性が高い |
